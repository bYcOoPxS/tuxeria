Configuración de Apache para Tuxeria

18 de diciembre de 2025

En este artículo explico cómo he configurado Apache para la web de Tuxeria. El objetivo principal era tener un servidor seguro, con HTTPS y autenticación básica.

Directorio principal

El DocumentRoot se ha establecido en /var/www/tuxeria. Para evitar que los usuarios vean archivos sensibles si el directorio no tiene un index.php, se ha deshabilitado el listado de directorios con Options -Indexes.

HTTPS y SSL

Se ha habilitado SSL en Apache con el certificado propio:

SSLCertificateFile /etc/ssl/certs/tuxeria.crt
SSLCertificateKeyFile /etc/ssl/private/tuxeria.key

Además, se fuerza la redirección de HTTP a HTTPS usando un VirtualHost en el puerto 80.

Autenticación básica

Para mayor seguridad se ha activado Basic Auth. Se utiliza un archivo de contraseñas:

/etc/apache2/.htpasswd_tuxeria

Esto obliga a cualquier usuario a autenticarse antes de acceder al contenido.

Resumen de directivas importantes

Options -Indexes +FollowSymLinks
AllowOverride All
Require valid-user
DirectoryIndex index.php index.html

Conclusión

Con esta configuración, la web de Tuxeria es accesible solo mediante HTTPS, con autenticación básica y sin posibilidad de listar directorios. Esto asegura un entorno más seguro y profesional.
